IEEE 802.1X

IEEE 802.1X is a network access control protocol that authenticates devices before allowing them to connect to a wired or wireless network.

In today\'s interconnected world, network security is more critical than ever. As shown in the following figure, 802.1X helps organizations ensure that no unauthorized users and devices can access resources within protected networks.

**Wired 802.1X**

On a wired network, open switch ports in conference rooms and other public areas within an organization can be attractive targets for intruders. Although it is a best practice to keep unused switch ports disabled until they are needed, managing port states in a large network can be administratively burdensome. As a result, unused ports may be left open.

One of the key benefits of 802.1X is that the authenticator, which is a switch in a wired network, and the authentication server manage the switch port's state. **Wireless 802.1X**

In wireless networks, user information is transmitted over the air, making it an easy target for attackers. To mitigate this risk, the 802.1X standard enables wireless technology manufacturers to implement strong authentication and authorization, enhancing security for enterprise networks against both internal and external threats. As a result, the Wi-Fi Protected Access (WPA) protocol was developed, particularly its Enterprise mode, which relies on the 802.1X authentication framework to ensure secure network access.

In a wireless 802.1X setup, either the access point (AP) or the wireless LAN controller (WLC) serves as the authenticator based on the network design. When the AP is the authenticator, it handles the 802.1X authentication directly with the client and communicates with the authentication server via RADIUS. When the WLC is the authenticator, the AP simply forwards the client's authentication packets to the WLC using CAPWAP, and the WLC manages the RADIUS communication and enforces policies. The choice depends on the deployment and configuration of the wireless network components.

EAPoW and CAPWAP are both related to wireless network security and management, but they serve different purposes. EAPoW is a security protocol that authenticates users connecting to a wireless network, while CAPWAP is a protocol that manages wireless access points (APs) and controllers.

**EAP Chaining**

Traditionally, endpoints could only authenticate either a user or a machine in the same EAP/RADIUS session, not both. To overcome this, EAP Chaining (supported in specific EAP methods) lets an authentication server authenticate both the user and the machine in a single authentication exchange. By authenticating both, EAP Chaining allows organizations to implement a Zero Trust Architecture (ZTA) as it verifies both the identity of the user and the legitimacy of the device.

RADIUS AVPs

A RADIUS AVP is a fundamental data structure that is used to convey information between a client and a server.

AVPs enable flexible and extensible communication by allowing a wide variety of authentication, authorization, and accounting parameters to be transmitted in a standardized format.

As illustrated in the following figure, each AVP consists of a type (attribute) field (which defines the type of information, such as username, password, or IP address), a length field (specifying the total size of the AVP), and a value (containing the actual data). RADIUS IETF attributes are a set of 255 standardized attributes that all RADIUS clients and servers must agree to.
