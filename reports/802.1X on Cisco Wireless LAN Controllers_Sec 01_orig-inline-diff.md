Cisco WLAN Components

Fundamentally, wireless networks consist of two Key Components: an AP and a WLC.

As shown in the following figure, an AP is a Layer 2 device that bridges 802.11 traffic to an 802.3 network and provides coverage within the wireless network. A WLC presents a central point for AP management, configuration, and optionally, user traffic termination.

![](media/image5.bin){width="6.940277777777778in" height="1.3074693788276466in"}

There are two types of Mandatory Access Control (MAC) implementations in a WLAN deployment: Local MAC and Split MAC.

  -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  MAC Implementation    Usage
  --------------------- -------------------------------------------------------------------------------------------------------------------------------------------------------------------
  **Local MAC**         All wireless MAC functions are performed at the AP. The complete 802.11 MAC functions, including management and control frame processing, are resident on the AP.

  **Split MAC**         MAC functions are divided between the AP and the controller. The AP performs real-time MAC functions while non-real-time functions are performed by the WLC.

**Choose an Authentication Mode**

When configuring 802.1X network authentication on a Windows client, it\'s important to select the appropriate authentication mode to determine how the device connects to the network. The chosen mode directly impacts the availability of network services during different stages of the login process.

Wireless authentication mode options include:

  ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  Authentication Mode                  Description
  ------------------------------------ --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  User or Computer authentication      Uses machine or user credentials. When a user is logged on, the user\'s credentials are used for authentication. When no user is logged on, machine credentials are used. This is the default setting.

  Computer authentication              Uses the device\'s machine credentials only, even after a user logs in. Useful for scenarios where device identity matters more than user identity.

  User authentication                  Uses only the user\'s credentials after login.
  ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Generally, it is recommended to choose the **User or Computer authentication** option. When using the option that authenticates only users, the machine cannot authenticate to the network before the user logs in. This option can cause issues for domain-joined machines needing early network access (e.g., for Group Policy, logon scripts, Active Directory contact).
